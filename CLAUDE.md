## Project Overview

Obsidian plugin for bidirectional synchronization between Plex Media Server (PMS) and Obsidian vault notes. Syncs watched status and media metadata. Written in TypeScript (~6,300 lines across 29 files). All work happens inside `plex-sync-obsdian-plugin/`.

## Commands

All commands run from `plex-sync-obsdian-plugin/`:

```bash
npm run build       # Production build (generates main.js via esbuild)
npm run dev         # Watch mode – rebuilds on file changes
npm run typecheck   # TypeScript type check (no emit)
npm run test        # Run unit tests (Vitest)
```

Run a single test file:
```bash
npx vitest run src/__tests__/sync-core.test.ts
```

After building, install to an Obsidian vault:
```bash
cp manifest.json main.js versions.json "/path/to/vault/.obsidian/plugins/plex-obsidian-sync/"
```

## Architecture

### Entry Point: `src/main.ts`

The `PlexObsidianSyncPlugin` class wires everything together:
- Registers Obsidian commands ("Sync Now", login, server refresh)
- Registers debounced file system hooks (modify: 450ms, delete: 1200ms) that trigger syncs
- Handles Plex OAuth PIN flow and persists settings (vault data API + localStorage for secrets)
- Pre-caches watched signatures before edits to detect changes

### Service Layer (`src/services/`)

| File | Responsibility |
|------|---------------|
| `sync-engine.ts` | Main sync orchestrator – acquires lock, fetches items, applies conflict policy, updates notes, syncs back to Plex |
| `sync-item-selection.ts` | Determines which items to fetch (account vs PMS mode) |
| `sync-pms-source.ts` | Fetches library items from PMS |
| `sync-note-paths.ts` | Resolves note file paths and frontmatter |
| `show-hierarchy-sync.ts` | Manages TV show/season/episode hierarchy structure |
| `sync-deletion-reconciliation.ts` | Handles items deleted from Plex |
| `plex-client.ts` | HTTP client for PMS XML API |
| `plex-tv-client.ts` | Plex.tv API client (OAuth, server discovery) |
| `plex-discover-client.ts` | Advanced account/library discovery |
| `vault-store.ts` | Obsidian vault file I/O (read/write/delete notes, parse frontmatter with `gray-matter`) |
| `logger.ts` | Debug logging with toggle |

### Core Business Logic (`src/core/`)

| File | Responsibility |
|------|---------------|
| `sync-core.ts` | Conflict resolution (`resolveConflictWinner`), metadata building (`buildManagedMetadata`), frontmatter merging (`mergeFrontmatter`) |
| `lock-core.ts` | Distributed locking via vault file (`.plex-sync-lock`) to prevent concurrent syncs |
| `plex-xml-parser.ts` | Parse PMS XML responses |
| `plex-tv-parser.ts` | Parse Plex.tv API responses |
| `plex-account-core.ts` | Plex account authentication logic |
| `constants.ts` | Managed frontmatter keys (38 keys with Portuguese/English aliases) |

### Key Architectural Patterns

**Conflict Resolution:** Three policies controlled by `ConflictPolicy = "latest" | "plex" | "obsidian"`. "latest" compares Plex timestamp vs note mtime.

**Frontmatter Management:** Two categories of keys – *managed* (38 keys, automatically updated, bilingual PT/EN) and *custom* (user-defined, preserved during sync). The `mergeFrontmatter` function in `sync-core.ts` enforces this separation.

**Authentication Modes:**
- `account_only` – Plex account token only, no PMS required
- `hybrid_account` – Account token + server discovery, selects specific PMS
- `manual` – Direct PMS URL + token

**Distributed Locking:** Critical for multi-instance Obsidian setups (e.g., LiveSync). Lock stored as vault file with TTL (default 30s).

**Event-Driven Sync:** No timers. Syncs triggered by: manual "Sync Now" command, or file system hooks (create/modify/delete).

**Bilingual UI:** Frontmatter keys have Portuguese/English aliases. Language detected from Obsidian locale at runtime, overridable in settings.
